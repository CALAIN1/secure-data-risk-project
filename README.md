# Secure Data & IA Responsable — Mini-projet d’analyse du risque



## Présentation du projet

Ce projet montre comment nettoyer, structurer et analyser des données relationnelles à l’aide de SQL.

L’objectif est de reproduire un workflow simple d’analyse :

- organiser des données brutes
- construire un modèle relationnel
- extraire des insights utiles



## Ce que j'ai réalise

- Structuration de données CSV en tables relationnelles
- Création d’un schéma SQL (clés primaires et étrangères)
- Rédaction de requêtes SQL pour analyser :
- les décisions selon le niveau de risque
- la distribution des prêts par secteur
- le profil des prêts refusés
- Création d’un fichier Excel de synthèse pour une lecture métier



## Structure du projet

- data_raw/ → données brutes (CSV)
- schema.sql → structure de la base de données
- queries.sql → requêtes d’analyse
- output/analysis.xlsx → synthèse des résultats


## Insights principaux

- Les décisions de crédit varient selon le niveau de risque
- Certains secteurs présentent plus de refus
- Les prêts refusés ont des caractéristiques communes



## SCompétences mises en avant

- Nettoyage et structuration de données
- SQL (jointures, agrégations)
- Analyse de données
- Traduction des données en insights métier

NOTE : Les données utilisées sont fictives et à des fins pédagogiques.
